## Overview
TrackMate is my senior capstone project — a MERN stack application for asset tracking and management. This was a 4-person team project for CS 4760. I was team lead. Grade: A.

## My Role
**Team Lead** — I wrote API endpoints and React components alongside one teammate who shared the bulk of the development work. Two other team members contributed less. I also set up the Docker Compose development environment and the seed data solution, which was my idea.

I did not build this solo. It was a team project and I want to be clear about that.

- **Stack**: MongoDB, Express.js, React, Node.js (MERN)
- **DevOps**: Docker, Docker Compose, environment management
- **Team Size**: 4 developers

**Repository Note**: This is a private repository under our professor's GitHub. Access: https://github.com/bradleypeterson/TrackMate.git

## What TrackMate Does
- Asset tracking dashboard with status updates
- Role-based access control (Admin, Manager, User)
- Audit trails for asset changes
- Search and filtering
- Mobile-responsive interface

## Problem
Live data was gated behind VPN. We needed a way to develop locally without VPN access, and team members needed identical environments to avoid integration issues.

## What I Built
- **Docker Compose** setup for orchestrating MongoDB, Node.js API, and React frontend
- **Seed scripts** with realistic JSON fixtures to replace VPN-gated data
- **Environment management** (.env files) for dev vs prod configurations
- API endpoints and React components (shared work with one teammate)
- Managed code reviews and sprint coordination via GitHub

```yaml
# docker-compose excerpt
services:
  api:
    build: ./api
    environment:
      - MONGO_URL=mongodb://db:27017/trackmate
```

## Results
- Team could work locally without VPN access
- Docker eliminated "works on my machine" issues
- Successfully deployed with proper security

## What I Learned
- Leading a team through a full development cycle
- Docker and container orchestration for team development
- RESTful API design with authentication and authorization
- MongoDB schema design
- The value of setting up good infrastructure early — it saved the team a lot of time
