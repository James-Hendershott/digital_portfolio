## Overview

A comprehensive, interactive 2,065-line single-page HTML application for planning and building a gravel pad foundation for a backyard shed. Built as a "Dad & James DIY Project" with zero external dependencies — no frameworks, no build tools, no npm. The entire application is a single self-contained HTML file.

## Key Features

- **7-Step Setup Wizard**: Customizes the entire guide based on pad size, gravel depth, sprinkler detection, drainage assessment, equipment choice, and existing tool inventory. Conditionally shows/hides relevant steps throughout all phases.
- **Satellite Property View**: Canvas 2D-based interactive tool that loads a property satellite image with distance measurement (2-click), area measurement (polygon drawing), adjustable scale calibration, and custom image upload support.
- **8 Construction Phases**: Detailed step-by-step guides (Planning → Prep → Mark → Excavate → Compact → Gravel → Move Shed → Final Setup) with interactive checklists, per-phase print buttons, and conditional content based on wizard answers.
- **7 Inline SVG Diagrams**: Hand-coded technical diagrams including 3-4-5 triangle layout, diagonal check, excavation cross-section, layer cross-section, deck block layout, shed-on-rollers side view, and skid build sheet.
- **Cost Calculator**: Live-updating calculator with cubic yard/tonnage calculations including waste factor, itemized breakdown (road base, top gravel, fabric, deck blocks, skids, rentals, tools), and KPI dashboard.
- **Materials & Where to Buy**: Tables with direct purchase links to Home Depot, Lowe's, and Amazon.
- **Print Support**: Full-page print stylesheet plus per-phase isolation printing with empty checkboxes.

## Architecture

The application is a zero-dependency monolith by design. All 2,065 lines live in one file — 267 lines of CSS, 474 lines of JavaScript, and 1,324 lines of HTML with inline SVG. The wizard state drives conditional rendering across all 8 phases, and the cost calculator recalculates live as inputs change. No state is persisted — everything runs in-memory during the session.
