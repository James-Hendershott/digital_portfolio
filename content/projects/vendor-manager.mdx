## Overview

This is a tool I built to fix real problems at my job as a Vendor Service Manager at Amerit Fleet Solutions. I track 40-100 fleet units across 6 food distribution centers, and the existing Smartsheet workflows were slow and manual. Nobody was going to fix that for me, so I started building something myself.

**What's honest here:** I designed the solution and identified every problem it solves. I built the early versions with hand-coded HTML forms. As it grew more complex — Smartsheet API integration, chart dashboards, AI document analysis, modals — I used Claude Code as my senior developer. I directed the architecture, verified the code, tested everything, and learned along the way. But I did not write most of the advanced features from scratch.

I use this tool every single day at work.

## The Dashboard (vendor-manager-web repo)

The original implementation. A browser-based tool that connects to Smartsheet's API, providing dashboards, workflow forms, and vendor communications.

**What I did:**
- Identified every workflow problem and designed the solution
- Built initial HTML forms and basic functionality by hand
- Directed all feature development and architecture decisions
- Tested everything against real work data daily for 2+ years
- Iterated based on actual workflow needs — this wasn't a school exercise

**What AI built (with my direction):**
- Smartsheet API integration with CORS proxy and caching
- Chart.js dashboards and drill-down reports
- Claude AI integration for estimate document analysis
- Complex modal systems and workflow form logic
- Email generation templates
- Dark/light theme system

**Scale:**
- ~20,000 lines across app.js, styles.css, and index.html
- 9 workflow input types, 3 report types, 6 chart drill-down reports
- Real-time Smartsheet sync with in-memory caching

## The MERN Rebuild (vendor-manager-mern repo — separate project)

I had AI scaffold a MERN stack rebuild of this tool, designed for team-wide deployment. This is a separate project from the dashboard. The backend architecture is substantially complete but the frontend is mostly placeholders. I plan to learn from this scaffold and eventually build a working version myself.

I'm not claiming I wrote the MERN codebase. It exists as a reference architecture that AI built to my specifications.

## Problem

As a Vendor Service Manager overseeing fleet maintenance across 6 locations, I was spending too much time on:
- Repetitive vendor emails with copy/paste across 40+ contacts
- Manual data entry and tracking across a 40-column Smartsheet
- No visibility into trends, aging units, or cost patterns
- No way to quickly detect units returned to service or past-due estimates

## How It Evolved

1. **Hand-coded HTML forms** — started simple, just trying to speed up email generation
2. **Added Smartsheet API** — connected to live data instead of manual entry (AI-assisted)
3. **Built dashboards** — Chart.js visualizations of fleet metrics (AI-assisted)
4. **Added AI features** — Claude API for document analysis and text improvement (AI-assisted)
5. **Daily iteration** — 2+ years of adding features based on real workflow pain points

## Results

- Used daily in production for 2+ years
- Every vendor interaction at my job flows through this tool
- Reduced time spent on repetitive communications
- Gave me visibility into fleet trends I didn't have before

## What I Learned

- How to identify real workflow problems and design solutions
- Working with APIs (Smartsheet, Claude) even when I didn't write every integration myself
- How a large codebase grows over time through daily iteration
- The difference between directing software development and writing every line yourself
- I'm planning to rebuild the dashboard from scratch in Python/Flask without AI as a learning exercise
